# Case Studies

Studies and zones help you manage delivery territories during peak seasons when order volumes increase. This feature ensures your logistics coverage matches real-world demand by allowing you to update zones regularly. You will achieve more efficient field operations by activating specific territory plans based on seasonality.

### Getting Started

Before creating studies, you must have the correct user permissions enabled.

*   System Access: Nomadia Delivery SaaS application.
*   Required Rights: **List of zones**, **Assign zones**, **Create and update zones**, **Access to the sectorization tool**, and **Delete zones and delete studies**.

1. Go to the **Configuration** module in the application banner.
![Frame at 1:51](../.images/Case Studies-Study creation with Primary Zone_timestamp_1_to_51.png "Configuration – The banner menu where you access system settings.")
2. Select the **Manage Users** page.
![Frame at 1:59](../images/Case Studies-Study creation with Primary Zone_timestamp_1_to_59.png "Manage Users – The list of configured users in the application.")
3. Edit the specific user to modify their access.
4. Click the **Roles and Rights** tab.
![Frame at 2:16](../images/Case Studies-Study creation with Primary Zone_timestamp_2_to_16.png "Roles and rights – The tab containing zone and study permissions.")
5. Check the necessary rights related to the zone and study modules.
6. Click the **Save** button to apply the rights.
![Frame at 3:00](../images/Case Studies-Study creation with Primary Zone_timestamp_3_to_00.png "Save button – Finalizes the permission changes for the user.")

### How To: Create a Study

1. Click the **Configuration** module.
2. Click the **Studies and zones** page in the delivery section.
3. Click the **Actions menu** and select **Create empty study**.
![Flow 3:29 to 3:44](../images/Case Studies-Study creation with Primary Zone_timestamp_3_to_29_to_3_to_44.gif "Create empty study – The sequence of opening the creation form.")
4. Enter the **Identifier**, **Name**, and associated **Agency**.
![Frame at 3:46](../images/Case Studies-Study creation with Primary Zone_timestamp_3_to_46.png "Study Form – The mandatory fields for naming and linking a study.")
5. Select a **Validity start date** and **Validity end date**.
6. Use the toggle to **Enable** the study.
7. Select the specific days of the week the study is active.
8. Define a specific seasonal activation window if needed.
![Frame at 4:36](../images/Case Studies-Study creation with Primary Zone_timestamp_4_to_36.png "Activation parameters – Setting the dates and days for seasonal use.")
9. Hit the **Save** button.

### How To: Add a Primary Zone

1. Open your study and switch to the **Zone tab**.
2. Open the **Actions menu** and select **Add a postal code zone**.
![Flow 5:38 to 5:44](../images/Case Studies-Study creation with Primary Zone_timestamp_5_to_38_to_5_to_44.gif "Add postal code zone – Opening the zone configuration form.")
3. Enter the **Identifier**, **Name**, **Agency**, and **Country**.
4. Set the **Postal code normalized length** (default is 5) and any required prefix.
5. Select **Primary zone** in the **Assignment mode** field.
![Frame at 6:26](../images/Case Studies-Study creation with Primary Zone_timestamp_6_to_26.png "Assignment mode – Selecting Primary zone to define a top-level territory.")
6. Enter postal codes one by one using the **plus button**.
7. For large regions, use the **Actions menu** and select **Import** to bulk upload data.
![Flow 6:57 to 7:07](../images/Case Studies-Study creation with Primary Zone_timestamp_6_to_57_to_7_to_07.gif "Import option – The process of bulk uploading postal codes.")
8. Click the **Save** button to visualize the zone on the map.

### Productivity Tips

- 💡 **Seasonal Activation**: Configure multiple activation windows within one year to handle different holiday campaigns.
- 💡 **Bulk Uploads**: Use the import tool for large lists of postal codes to save time when setting up entire regions.
- ⚠️ **Assignment Mode**: Always select **Primary zone** in the assignment mode field to ensure the system recognizes it as a top-level zone rather than a subzone.
