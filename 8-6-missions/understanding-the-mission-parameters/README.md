# Understanding the Mission Parameters

### Understanding the mission parameters

Mission Parameters are the core data elements that define the characteristics, objectives, and execution details of each mission. Understanding these parameters is essential for ensuring that missions are properly configured, assigned, and executed in the field.

These parameters provide both operational and logistical information used by

coordinators, dispatchers, and deliverers.

#### 1.4.1. Access Mission Parameters

From the Mission page

1. Click the pencil icon next to the mission you want to edit

#### Mission parameters description

## Mission Parameters Tab

| Section     | Parameter                           | Description                                                                   |
| ----------- | ----------------------------------- | ----------------------------------------------------------------------------- |
| **Mission** | Barcode                             | Automatically generated by Nomadia Delivery to uniquely identify the mission. |
|             | Agency (Mandatory)                  | Mandatory field specifying the agency responsible for handling the mission.   |
|             | Subcontractor                       | Third-party delivery service provider assigned to complete the mission.       |
|             | Sender Barcode                      | Optional barcode supplied by the contractor.                                  |
|             | Parent Container ID                 | Identifies the main container or group to which the mission belongs.          |
|             | Grouping Key                        | Groups missions linked to the same customer for easier handling.              |
|             | Container Type                      | Type of container used (box, pallet, crate, etc.).                            |
|             | Origin                              | Original dispatch location of the delivery.                                   |
|             | Scheduled Deliverer                 | Name of the assigned delivery agent.                                          |
|             | Priority                            | Indicates urgency (High, Medium, Low).                                        |
|             | Status Configuration Type           | Defines the workflow applied to the mission.                                  |
|             | One Type of Password                | Security code (PIN/OTP) used for verification.                                |
|             | Mission Status                      | Current mission state (e.g., Waiting, Delivered).                             |
|             | Sub-status                          | Detailed state within the main status.                                        |
|             | Motive                              | Reason for the current sub-status.                                            |
|             | Mission Type                        | Delivery, Pickup, Chained Pickup & Delivery, Visit, Drop Shipping.            |
|             | Provisional Date                    | Tentative mission date.                                                       |
|             | Main Depot Scan Date                | Date/time when scanned at the depot.                                          |
|             | Storage Place                       | Location within the depot where the item is stored.                           |
|             | Fixed Visit Duration                | Estimated time (in minutes) to complete the mission.                          |
|             | Compatibility with Resources        | Indicates compatibility with assigned resources.                              |
|             | Require All Skills to be Compatible | Ensures resource meets all required skills.                                   |
|             | Delivery Before Pickup              | Enforces delivery before pickup sequence.                                     |
|             | Keys                                | Indicates if keys/access codes are required.                                  |
|             | Position                            | Order of the mission in the route.                                            |

### Parcel

| Section    | Parameter     | Description                                             |
| ---------- | ------------- | ------------------------------------------------------- |
| **Parcel** | Product Type  | Category of the product (e.g., electronics, groceries). |
|            | Length        | Package length.                                         |
|            | Width         | Package width.                                          |
|            | Height        | Package height.                                         |
|            | Weight        | Package weight.                                         |
|            | Volume        | Package volume.                                         |
|            | Package Value | Declared monetary value.                                |
|            | Comment       | Internal notes for the mission.                         |

### Pickup

| Section    | Parameter                     | Description                                   |
| ---------- | ----------------------------- | --------------------------------------------- |
| **Pickup** | Address (Mandatory)           | Pickup address (non-editable after routing).  |
|            | Contact Details               | Sender’s designation, name, phone, and email. |
|            | Picked Up By                  | Person/agent who performed the pickup.        |
|            | Pickup Asked Date / End Date  | Requested pickup time window.                 |
|            | Scannable Container Code      | Barcode for identifying the container.        |
|            | Pickup Instructions / Comment | Special instructions for pickup.              |

### Delivery

| Section      | Parameter                       | Description                                      |
| ------------ | ------------------------------- | ------------------------------------------------ |
| **Delivery** | Address (Mandatory)             | Delivery address (non-editable after routing).   |
|              | Contact Details                 | Recipient’s designation, name, phone, and email. |
|              | Delivered By                    | Person/agent who completed delivery.             |
|              | Delivery Asked Date / End Date  | Requested delivery time window.                  |
|              | Cash on Delivery                | Amount to be collected.                          |
|              | Delivery Price                  | Delivery cost for billing.                       |
|              | Delivery Instructions / Comment | Special delivery instructions.                   |
