# intractingwithiframe

This feature allows you to communicate directly with subcontractors through an integrated external interface. You can send specific status updates and messages without leaving the main application. This ensures seamless data passing between your system and the subcontractor's external portal.

### Getting Started

*   Access to the **Machines** page.
*   A pre-configured subcontractor URL set in your system settings.

1.  Navigate to the **Machines** page.
2.  Select the desired **Machine** from the list.

    ![Frame at 0:08](../images/intractingwithiframe-interactingwithiframe_timestamp_0_to_08.png "Machines List – A specific machine is selected in the main grid.")

3.  Click on **Actions**.

    ![Flow 0:08 to 0:12](../images/intractingwithiframe-interactingwithiframe_timestamp_0_to_08_to_0_to_12.gif "Actions Menu – The user clicks the Actions button to reveal options.")

4.  Click on **Assign to subcontractor**.

    ![Flow 0:12 to 0:24](../images/intractingwithiframe-interactingwithiframe_timestamp_0_to_12_to_0_to_24.gif "Assign to Subcontractor – The integrated subcontractor interface opens.")

### Feature Overview

*   **Message field**: Use this area to type custom instructions or status updates.

    ![Frame at 0:32](../images/intractingwithiframe-interactingwithiframe_timestamp_0_to_32.png "Message Field – The text input area for subcontractor messages.")

*   **City field**: Displays or allows for the entry of specific location data.

    ![Frame at 0:32](../images/intractingwithiframe-interactingwithiframe_timestamp_0_to_32.png "City Field – The field designated for city information.")

*   **Actions**: Selectable status categories like **Info**, **Warning**, **Error**, or **Success**.

    ![Frame at 0:37](../images/intractingwithiframe-interactingwithiframe_timestamp_0_to_37.png "Status Selectors – Radio buttons used to define the nature of the message.")

*   **Send Message**: Submits your data and triggers a confirmation popup in the interface.

    ![Frame at 0:47](../images/intractingwithiframe-interactingwithiframe_timestamp_0_to_47.png "Send Message Button – The button used to transmit the message.")

### How To: Send Status Communications

**Sending an Info Update**
1.  Type your message in the **Message field**.
2.  Select the **Info** status.

    ![Flow 0:32 to 0:47](../images/intractingwithiframe-interactingwithiframe_timestamp_0_to_32_to_0_to_47.gif "Sending Info – User types a message and selects Info status.")

3.  Click **Send Message**.
4.  Review the "subcontractor has been passed" popup.

    ![Frame at 0:47](../images/intractingwithiframe-interactingwithiframe_timestamp_0_to_47.png "Info Popup – Confirmation popup for an info-level update.")

**Sending a Warning**
1.  Edit the existing text in the **Message field**.
2.  Change the status to **Warning**.

    ![Flow 0:55 to 1:13](../images/intractingwithiframe-interactingwithiframe_timestamp_0_to_55_to_1_to_13.gif "Setting Warning – User modifies the message and selects Warning.")

3.  Click **Send message only**.
4.  Review the "Check the email once again" popup.

    ![Frame at 1:13](../images/intractingwithiframe-interactingwithiframe_timestamp_1_to_13.png "Warning Popup – Alert popup regarding email verification.")

**Sending an Error or Phone Number**
1.  Enter the required details in the **Message field**.
2.  Select the **Error** status.

    ![Flow 1:23 to 1:35](../images/intractingwithiframe-interactingwithiframe_timestamp_1_to_23_to_1_to_35.gif "Setting Error – User enters data and selects Error status.")

3.  Click **Send message only**.
4.  Review the phone number popup.

    ![Frame at 1:35](../images/intractingwithiframe-interactingwithiframe_timestamp_1_to_35.png "Error Popup – Popup displaying the entered phone number.")

**Finalizing a Successful Assignment**
1.  Select the **Success** status.
2.  Edit the final **Message**.

    ![Flow 1:55 to 2:10](../images/intractingwithiframe-interactingwithiframe_timestamp_1_to_55_to_2_to_10.gif "Setting Success – User prepares final success communication.")

3.  Click **Send message only**.
4.  Review the "subcontractor has been successful" popup.

    ![Frame at 2:15](../images/intractingwithiframe-interactingwithiframe_timestamp_2_to_15.png "Success Popup – Final confirmation of a successful subcontractor action.")

**Exiting the Interface**
1.  Refresh the page once tasks are complete.
2.  Close the popup.

    ![Flow 2:19 to 2:30](../images/intractingwithiframe-interactingwithiframe_timestamp_2_to_19_to_2_to_30.gif "Exiting – User refreshes and closes the interface to return to the main view.")

### Productivity Tips

*   💡 **Return to Missions**: Closing the final popup automatically returns you to the **Missions** page.
*   ⚠️ **Confirm Action**: Always click **Send Message** to trigger the required status popup and ensure the update is processed.
